Web Proposal Code Sequence

Before properly doing my website code - I had created a simple format to entail how to code the website proposal in an efficient and productive manner. The format included of making my basic format for the one-page scroll first (via adding appropriate div sections to split up the content to fit on 1 page size) to appropriately get the basic layout, then moving towards developing the individual pages following the basic formatting (e.g. working with the home and nav bar and then moving onto the appropriate other pages) and also doing mobile pages incrementally as each page is done. For instance when the home page is done - make it responsive to mobile versions - via use of media queries and referring to the mobile consideration images from the last section.

Whilst I followed majority of this stage - I had however failed to compromise with the mobile format/other resolution size formats later on as it became too tricky to implement given time constraints and the lack of autonomous and responsive code within the HTML formatting. As such for the future - it would be better to consider this beforehand - and perhaps experiment on individiual pages before moving onto the normal one-apge scroll section

Flex Box Process for Our Impact and Your Impact Pages

The use of CSS Flexbox was an essential and critical component towards the development of my "Our Impact" and "Your Impact" pages - they were aligned in different positions and required proper div tags and also flexbox css properties in order to align them accordingly (as I wanted on my mockup designs). Before the use of Flexbox, I had a hard time properly aligning them and found it impossible to do - and the discovery it had just made things much much easier, with the tool being very powerful and useful in properly aligning the images and textual elements accordingly. For instance - for my "Your Impact" page - I had used according column divs - housing row divs (as shown in the following images below) in order to align them properly, using the flex-direction CSS property to properly align them to column or row elements as such . Additional Changes and Additions

Additional added elements were also added to my website - including the use of simple bottom hover animations alongside a footer showing World Vision's contact details instead of an image carousel - to which helped show their contact details at all times instead of being exclusive to the home page area due to the footer being fixed to the bottom section as the user scrolls bar. The addition of a button hover option was according to not only my own judgement initally but also from user feedback (as shown more in the self-reflection section) whereby suggestions towards adding it to make the website more dynamic and interesting were done. However - looking back, I should've also done more animations such as little image hover bobbles or nav bar animations as a way to be more creative and less dynamic - however I'm relatively happy about the button hover animations and also interaction with the drop-down menu and input fields for the donation-form. The footer was also a nice addition - linking to World Vision's social medias via the use of font-awesome icons alongside text detailing their contact numbers as well. Self-Reflection and Considerations

Overall, I'm relatively happy with the outcome of my website proposal/prototype - given my time constraints alongside how I was able to accordingly follow majority of my design aesthethics and layout given the mockups - to which were useful by the end of it. I was also happy about the process towards coding it - ie doing the overall basic format first and then coding the individual pages and other responsive sizes accordingly. However - the main issue to which occurred was the lack of responsive design - to which was something addressed in user feedback but couldn't be properly addressed due to time constraints alongside the difficulty I had in managing my HTML/CSS code which could've been planned and managed more better and productively via experimenting or doing them on other pages before combining them altogether.
